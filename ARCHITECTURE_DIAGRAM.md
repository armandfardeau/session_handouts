# Category Pages Architecture

```
┌─ MAIN CATEGORIES PAGE ─────────────────────┐
│  /categories.html (manually edited)        │
│  Shows: Grid of all categories + counts    │
│  Links to: /category/{slug}/               │
└────────────────────────────────────────────┘
          │
          │ (user clicks category card)
          ↓
┌─ INDIVIDUAL CATEGORY PAGES ────────────────┐
│  /category/{slug}/index.html               │
│  (AUTO-GENERATED by plugin at build time)  │
│  Layout: _layouts/category.html            │
│  Data: Filtered posts from that category   │
│  Links to: individual posts                │
└────────────────────────────────────────────┘
          │
          │ (user clicks post card)
          ↓
┌─ POST DETAIL PAGE ─────────────────────────┐
│  /post-title/index.html                    │
│  Layout: _layouts/post.html                │
│  Shows: Full article + metadata            │
│  Links to: category pages                  │
└────────────────────────────────────────────┘

PLUGIN LOGIC (Ruby, runs at build time)
========================================

1. Jekyll.generate() event fires
2. Plugin iterates site.posts
3. Extracts unique category names
4. For each category:
   - Create slug: lowercase + replace spaces with dashes
   - Create Page object (not tied to a file on disk)
   - Set layout: "category"
   - Set data: category name, title, permalink
   - Add to site.pages
5. Jekyll renders each generated page using category.html layout


LIQUID FILTERING (renders on page)
==================================

category.html uses:
```liquid
{% assign category_posts = site.posts | where_exp: "post", "post.categories contains page.category" %}
{% for post in category_posts %}
  {% include main-loop-card.html %}
{% endfor %}
```

This filters posts to only those containing the current page's category.
